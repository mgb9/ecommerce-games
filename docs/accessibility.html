Accessibility statement

For the WM956-15 e-commerce learning games at mgb9.github.io/ecommerce-games — four browser-based teaching simulations and their hub page.

Before this statement is published for students, the following must be completed by the module owner: the accessibility contact address, the enforcement/escalation route confirmed against the University of Warwick’s central accessibility statement, and the dates below. These are marked [TO CONFIRM] and must not be left as placeholders on a live public-sector service.

Using this website

These games run entirely in your browser. We want as many people as possible to be able to use them. You should be able to:

  • navigate the hub and all four games using a keyboard alone;
  • see a clear focus indicator on every interactive control, on both the dark game screens and the light hub;
  • skip repeated page furniture using the “Skip to main content” link;
  • switch on Simpler English to replace idiomatic wording with plainer wording throughout;
  • open a plain-English definition for any specialist term shown with a dotted underline;
  • zoom to 200% and have text reflow without loss of content;
  • have animations reduced automatically if your device requests reduced motion.

How accessible this website is

These games are partially compliant with the Web Content Accessibility Guidelines version 2.2 AA standard, because of the non-compliances listed below. They are interactive data-analysis simulations, and some of their content — particularly charts and live simulation output — is not yet fully available to assistive technology.

Non-accessible content

The content below is non-accessible for the following reasons.

Non-compliance with the accessibility regulations

IssueWCAG criterionPlanned fix
Charts in Marketplace Tycoon, Conversion Lab and Funnel Fixer are drawn as graphics with no text alternative or accompanying data table, so their content is unavailable to screen-reader users. (Data Detective’s segment reports are already real data tables.) 1.1.1 Non-text Content (A) Add an accessible summary and a toggleable data table beneath each chart. [TO CONFIRM date]
Chart series are distinguished by colour alone (for example control vs variant, and competitor stores). 1.4.1 Use of Colour (A) Add non-colour encoding (line style, direct labels) alongside the data tables. [TO CONFIRM date]
Inside the games, many visual section headings are styled text rather than semantic headings, so screen-reader users cannot navigate by heading. 1.3.1 Info and Relationships (A) Convert in-game section titles to semantic headings. [TO CONFIRM date]
Results that appear after an action — test verdicts, weekly simulation outcomes, diagnosis scoring — are not announced to screen readers. 4.1.3 Status Messages (AA) Add live regions to result panels. [TO CONFIRM date]
Each game keeps its own header directly beneath the shared WMG bar, and both stick to the top of the screen. When the page is scrolled they overlap, which can reduce the clickable area of the bar’s “All games” link. The link remains operable, but the overlap should be removed. 2.5.8 Target Size (Minimum) (AA) Rework the two stacked sticky headers so they do not overlap. [TO CONFIRM date]
The glossary definition popovers open on click but are not exposed as expandable controls to assistive technology. 4.1.2 Name, Role, Value (A) Rebuild as proper disclosure buttons. [TO CONFIRM date]
Some dense screens (the Data Detective dashboard and the Conversion Lab wireframe studio) have not been verified for reflow at 320 CSS pixels. 1.4.10 Reflow (AA) Responsive audit and fixes on small viewports. [TO CONFIRM date]

Content not yet assessed

  • The games have not yet been tested with screen readers (NVDA, JAWS or VoiceOver), nor with speech-recognition software. Automated and manual code-level checks only have been carried out so far, and automated tools detect only a minority of accessibility problems.
  • No testing has been carried out with disabled users.

Third-party content

  • Web fonts are loaded from Google Fonts (fonts.googleapis.com). This means your IP address is sent to Google when a page loads. No other data is transmitted: all game logic, scoring and saved preferences stay in your browser.

What we have already fixed

  • Text and interface colours were measured against WCAG AA; seven failing colour pairs were corrected (de-emphasised text on the game panels, and three category labels on the hub cards).
  • A visible focus indicator was added that meets the 3:1 non-text contrast requirement on both the dark game screens and the light hub.
  • A “Skip to main content” link and a <main> landmark were added to every page.
  • Reduced-motion and Windows High Contrast support were added.
  • Hub headings were given correct semantics and decorative numerals were hidden from assistive technology.
